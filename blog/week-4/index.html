ARRAYS AND HASHES

JANUARY 2, 2016

So this is week four of Dev Bootcamp's phase 0 and it's the first time we're diving into Ruby. I gotta say that I'm feeling pretty good about it even after skipping a full two weeks for the holidays. Jumping back in felt pretty natural save for a few hiccups with my memory of CSS during GPS 2.1.

This post is going to talk about arrays and hashes, what they are and how they differ. My understanding of this topic is surface level at best, but I'll try to convey what knowledge I do know and provide resources for further reading at the end.

Both arrays and hashes are containers of information. Arrays can be thought of as a simple list that go in a particular order. Each entry counts up an index starting from 0. Here's an example of an array:


					      my_first_array = ["dog","cat","bird","whale"]
					      think of it like this...
					      my_first_array = [0,1,2,3]

The "dog" entry is index 0 and corresponds to my_first_array[0], "cat" is index 1 and corresponds to my_first_array[1], and so on... If we were to add another entry to this array then that entry would be index 4. Programmers generally begin counting from zero. Arrays can hold pretty much any data type like strings, integers, floats, and even other arrays.

Hashes on the other hand can be thought of as pairs of information called key-value pairs with each pair having a name (key) and a value. Here's an example of a hash:


					      my_first_hash = {
					      :key0 = "dog"
					      :key1 = "cat"
					      :marko = "bird"
					      :key3 = "whale"
					    }

Unlike arrays in which the values correspond to index numbers, hashes have keys that correspond to values. The keys can be anything that has relevant meaning to the programmer. I'm fairly certain that that's the point of using hashes. It's a way to create, access, and remember information easily. Values can be accessed by calling their key.


					      return my_first_hash(:marko)

This would return "bird". And just like arrays, hashes can contain pretty much any data type from strings, integers, floats, arrays, and even other hashes.
