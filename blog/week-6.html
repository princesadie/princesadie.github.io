CLASSES IN RUBY

JANUARY 16, 2016

Definition from the doc.

Classes in Ruby are first-class objects---each is an instance of class Class.

I like to think of classes as blueprints from which many objects can be modeled. For example, take a Class Animal with @brain, @number_of_legs as an instance variable and move and still as methods. 

				Class Animal
					def initialize(num_of_legs)
						@brain = true
						@number_of_legs = num_of_legs
						@speed = 0
					end

					def move
						if @number_of_legs <= 0
							@speed = 1
						else
							@speed = 2
						end
					end

					def breathe
						if @brain = true
							p "I can breathe!"
						end
					end
				end

I think of instance variables as attributes/characteristics that every object created from the class automatically gets and is accessible by every method in the class. Think of methods as the actions that the object can do, in this case our Animal can move and breathe. 

					human = Animal.new(2)
					human.breathe => "I can breathe!"

The above code would create a human from the Animal class and it would initialize with two legs. What if we wanted a four legged pig? Coming right up! 

					pig = Animal.new(4)
					pig.breathe => "I can breathe!"

It's as simple as that. Once we define our class, we can create as many different objects from that class as we want. We can even make something ridiculous like a horse with 10 legs. 

					horse = Animal.new(10)

We can even take it a step further and create a whole new subclass underneath our Animal class and instill it with a new set of variables. 

				  Class Pet < Animal
				  	def initialize(num_of_legs)
				  		@num_of_legs = num_of_legs
				  	end

				  	def play
				  		puts "I can play!"
				  	end
				  end

				  dog = Pet.new(4)
				  dog.breathe => "I can breathe!"
				  dog.play => "I can play!"

Can you see that our new Pet subclass has only one method play? Our new dog can still breathe despite us not writing that method out in this new subclass because it inherits the "ability" or method from it's parent class Animal.

That was a super rough explanation of Classes in Ruby. Take a look at Stackoverflow and the docs for a wealth of information and awesome explanations for further reading!
