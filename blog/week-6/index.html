Classes in Ruby

JANUARY 16, 2016

Definition from the doc.

Classes in Ruby are first-class objects---each is an instance of class Class.

I like to think of classes as blueprints from which I can create many objects can be modeled. For example, take a Class Animal with @brain, @number_of_legs as an instance variable and move and still as methods. 

						Class Animal
							def initialize(num_of_legs)
								@brain = true
								@number_of_legs = num_of_legs
							end

							def move
							end

							def still
							end
						end

I think of instance variables as attributes/characteristics that every object created from the class automatically gets. Think of methods as the actions that the object can do, in this case our Animal can move and breathe. 

						Class Animal
							def initialize(num_of_legs)
								@brain = true
								@number_of_legs = num_of_legs
							end

							def move
							end

							def breathe
							end
						end

						Class Human < Animal
							def initialize(num_of_legs)
								@brain = true
								@number_of_legs = num_of_legs
							end
						end

						Class Pet < Animal
							def initialize(num_of_legs)
								@brain = true
								@number_of_legs = num_of_legs
							end
						end

						person = Human.new(2)
						dog = Pet.new(4)
						snake = Pet.new(0)

The above Class Human is a subclass of Animal and the line person = Human.new(2) would create a new object from Human with 2 legs.

Instance variables allow to use their values throughout the class unlike local variables. If we were to use local instead of instance, it'd look something like below.
