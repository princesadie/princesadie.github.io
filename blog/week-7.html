LOOPING IN RB & JS

JANUARY 23, 2016

A quick review...

Looping in programming is invoking a series of instructions or actions repeatedly until a certain condition is met. In general it can be seen like this, data is provided, this data is modified or used/acted upon in some way, and then a condition is checked. If the condition is met, then the loop ends there. If the condition is not met, then the loop repeats. What if the condition is never met and the loop keeps on repeating? Over and over and over and over... You just created an infinite loop. An infinite loop, as the name suggests, is a never ending loop, and is something you never want to create. It'll probably go on looping until your system gives up and calls it quits by crashing.

In Ruby...

Ruby has quite a few methods we can easily use to loop. For instance, if I wanted to loop through an array of names and print those names out on the screen, I could do it using the following snippet. 

	list_of_names = ['Prince','Michelle','Ronin','Saya','John','Nixon','Jim']

	list_of_names.each do | names |
		puts names
	end

	=> 'Prince'
	=> 'Michelle'
	=> 'Ronin'
	=> 'Saya'
	...so on...

A super easy solution to a simple task. But how about in Javascript?

In Javascript...

Javascript is still quite new to me at this point so forgive any missteps after this point. Looping in Javascript isn't quite so simple as in Ruby. There are some built-in functions that can assist, but I'll demonstrate the way I've learned it. In Ruby the '.each' method easily steps through each element in the given array and then performs the action in the proceeding block. In Javascript we've got use a 'for' loop. A 'for' loop looks like the snippet below. 

	for ([initialization]; [condition]; [final-expression])
			statement

The [initialization] part is generally where a counter variable is initialized. The [condition] part is what's evaluated before each loop occurs. If [condition] returns true, then the loop proceeds and statement is executed. If [condition] returns false, then the loop is not executed and we skip to the next expression in the program.

So here's what our Ruby example above would look like in Javascript. 

	var list_of_names = ['Prince','Michelle','Ronin','Saya','John','Nixon','Jim'];

	for (var counter = 0; counter < list_of_names.length; counter++) {
		console.log(list_of_names[counter]);
	}

In plain english...

The var counter = 0 initializes our counter variable.

counter < list_of_names.length checks to see if counter is less the length of our array. If it is, then we can proceed to the statement which in this case is console.log(list_of_names[counter]).

counter++ adds one to our counter after every iteration. Once this counter is greater than the length of list_of_names, then our condition is no longer true and the loop is terminated.

Here's what my pseudocode would look like. 

	CREATE array of names 'list_of_names'
	START for loop
		INITIALIZATION
			CREATE variable 'counter' equal to 0
		CONDITION
			CHECK IF 'counter' is less than length of 'list_of_names'
		FINAL-EXPRESSION
			ADD one to 'counter'
		STATEMENT
			PRINT each element of 'list_of_names' based on 'counter'

Hopefully that helped in some way! Take a look at the MDN on for loops here.
